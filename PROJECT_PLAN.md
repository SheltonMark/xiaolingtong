# 小灵通微信小程序 - 项目开发计划表

## 📋 项目概览

**项目名称**: 小灵通 - 工厂贸易供需对接平台
**技术栈**: 微信原生小程序 + Node.js + MySQL + Redis
**项目规模**: 大型（72个API接口，30+张表，6大核心模块）
**开发模式**: 双端开发（小程序用户端 + Web管理后台）

---

## 🎯 核心业务场景

| 业务场景 | 需求方 | 服务方 | 核心功能 |
|---------|-------|-------|---------|
| 外贸采购对接 | 贸易公司 | 工厂 | 发布采购需求，工厂查看并联系 |
| 工厂库存对接 | 电商/贸易公司 | 工厂 | 发布库存信息，渠道方采购 |
| 临时用工对接 | 工厂 | 临工 | 完整用工闭环（发布→报名→监管→结算） |
| 代加工服务对接 | 需求方 | 有产能工厂 | 发布代加工需求，工厂承接 |

## ⚠️ 7大技术难点

| 难点 | 描述 | 解决方案 |
|-----|------|---------|
| 1. 双用户体系权限隔离 | 企业和临工看到的页面完全不同 | 路由层面权限控制，前端组件按用户类型渲染 |
| 2. 临工用工状态机 | 涉及多方确认，状态流转复杂 | 设计严格的状态机，每个状态转换都有明确条件 |
| 3. 微信支付集成 | 企业付款、用户提现、订单支付 | 使用微信支付API，处理回调和异常情况 |
| 4. 信用分自动计算 | 基于行为自动触发 | 设计触发器机制，在关键操作后自动计算 |
| 5. 查看机会消耗 | 防重复查看，会员判断 | Redis缓存 + 数据库记录，FIFO过期策略 |
| 6. 实时通知推送 | 双渠道推送，订阅授权 | 站内消息必发，微信通知需授权，定时任务处理 |
| 7. 地理位置计算 | 用工列表按距离排序 | 使用经纬度计算距离，数据库存储坐标 |

## 💰 商业模式

### 收费项目
1. **会员费**: 月会员/年会员，无限次查看联系方式
2. **查看机会**: 非会员查看联系方式消耗次数
3. **广告位**: 首页Banner、推荐位，按天计费
4. **信息置顶**: 发布的信息置顶展示
5. **用工抽成**: 工厂设定25元/小时 → 平台抽成20% → 临工看到20元/小时

### 配置原则
**所有价格、比例、次数、天数等参数必须后台可配置，代码中不写死任何数值**

## 📝 重要设计原则

### 1. 所有数值后台可配置
**绝对不要在代码中写死任何数值**，包括：
- 价格（会员价格、查看机会价格、广告位价格、置顶价格）
- 次数（新用户初始查看机会、有效期天数）
- 比例（平台抽成比例、管理员服务费分成比例、超额报名比例）
- 列表（开放城市、工种列表、关键词黑名单）

### 2. 用户类型隔离
- 企业用户和临工用户看到的页面、搜索范围完全不同
- 这是防止商业模式漏洞的核心设计

### 3. 信息与联系方式分离
- 信息本身（标题、描述、图片）免费浏览
- 联系方式需要消耗查看机会或会员权益
- 临工查看用工信息完全免费（包括联系方式）
