## 📖 使用指南

### 新成员入职
1. 阅读 [研发管控总览](./DEVELOPMENT_PLAN.md)
2. 阅读 [需求分析文档](./findings.md)
3. 根据角色阅读对应的架构设计文档
4. 学习 [编码规范](./CODING_STANDARDS.md)

### 开始开发
1. 查看 [任务清单](./TASK_CHECKLIST.md) 认领任务
2. 参考 [模块详细拆解](./MODULE_BREAKDOWN.md) 了解实现细节
3. 参考 [API接口规范](./API_SPEC.md) 进行前后端对接
4. 遵循 [编码规范](./CODING_STANDARDS.md) 编写代码

### 测试阶段
1. 参考 [测试计划](./TESTING_PLAN.md) 编写测试用例
2. 执行测试并记录Bug
3. 使用 [进度跟踪模板](./PROGRESS_TRACKING.md) 跟踪Bug

### 部署上线
1. 参考 [部署指南](./DEPLOYMENT_GUIDE.md) 进行部署
2. 执行 [上线评审会议](./MEETING_TEMPLATES.md#上线评审会议)
3. 准备回滚方案

### 项目管理
1. 每日使用 [进度跟踪模板](./PROGRESS_TRACKING.md) 记录进度
2. 每周召开 [项目周会](./MEETING_TEMPLATES.md#项目周会)
3. 定期更新 [风险管理](./RISK_MANAGEMENT.md)
4. 项目结束后召开 [复盘会议](./MEETING_TEMPLATES.md#复盘会议)
