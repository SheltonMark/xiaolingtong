# Findings & Decisions - 小灵通微信小程序

## Requirements

### 核心需求
- 微信原生小程序开发
- 双用户体系：企业用户 vs 临工用户（完全隔离）
- 4大业务场景：外贸采购、工厂库存、临时用工、代加工服务
- 后端：Node.js + MySQL + Redis
- 管理后台：独立Web后台（PC端）
- 支付：微信支付（会员、查看机会、工资发放、提现）

### 功能需求
- 用户注册登录（仅微信授权）
- 企业认证（营业执照+法人身份证）
- 临工认证（身份证+手机号）
- 信息发布与审核
- 搜索筛选功能
- 联系方式查看（消耗机会/会员免费）
- 临工用工完整闭环（发布→报名→分配→打卡→结算→支付→提现）
- 信用分系统（自动计算）
- 消息通知（站内+微信服务通知）
- 管理后台（审核、配置、统计）

### 完整模块分析

#### 1. 供需信息模块
- **3种信息类型**: 采购需求、工厂库存、代加工服务
- **关键词黑名单检查**: 发布时自动检测,触发后保留原文提示修改
- **审核模式**: 支持自动审核/人工审核切换
- **查看联系方式**: 会员免费,非会员消耗查看机会,同一信息不重复扣费
- **信息置顶**: 付费推广功能,多个时长套餐
- **举报功能**: 用户可举报虚假信息,管理员处理

#### 2. 临工用工模块 (最复杂)
- **角色**: 工厂、临工、临时管理员、平台管理员
- **抽成机制**: 工厂设定25元/小时 → 平台抽成20% → 临工看到20元/小时
- **超额报名**: 允许报名人数 = 需求人数 × (1 + 比例),默认50%
- **出勤确认**: 开工前24小时提醒,12小时截止,未确认释放名额
- **临时管理员**: 从报名者中指定,需满足接单≥10单、信用分≥95分
- **监管职责**: 签到、打卡、工时/计件记录、现场照片上传、异常上报
- **结算流程**: 管理员提交核验单 → 临工确认 → 工厂支付 → 自动分账
- **管理员服务费**: 从平台收入中支出,不额外向工厂收取

#### 3. 支付与财务模块
- **5大收费项目**: 会员费、查看机会、广告位、信息置顶、用工抽成
- **会员体系**: 月会员/年会员,无限次查看联系方式
- **查看机会**: 新用户赠送、购买获取,有效期7天,FIFO消耗
- **广告位**: 首页Banner、首页推荐位,按天计费,需审核
- **工资支付**: 工厂支付 → 平台分账 → 临工余额 → 提现到微信零钱
- **提现规则**: 随时提现,免手续费,即时到账

#### 4. 消息与通知模块
- **双渠道**: 站内消息(必发) + 微信服务通知(需授权)
- **消息分类**: 系统通知、业务消息
- **推送时机**: 实时推送(审核结果、工资到账) + 定时推送(出勤提醒、到期提醒)
- **订阅消息模板**: 审核结果、报名结果、订单状态、到账通知、提醒通知
- **在线客服**: 使用微信小程序客服能力

#### 5. 管理后台模块
- **8大功能模块**: 审核中心、用户管理、用工管理、财务管理、内容管理、数据统计、系统配置、诚信曝光
- **审核功能**: 信息审核、认证审核、广告审核、举报处理
- **用工管理**: 人员分配、指定临时管理员、修改抽成比例
- **财务统计**: 收入统计(5项)、支出统计(3项)、交易明细
- **系统配置**: 所有价格、比例、次数、天数等参数后台可配
- **诚信曝光**: 曝光榜管理,小程序端展示

#### 6. API接口设计
- **接口总数**: 72个
- **模块分布**: 用户(7) + 供需信息(10) + 临工用工(16) + 支付财务(9) + 消息(6) + 公共(3) + 管理后台(21)
- **响应格式**: 统一JSON格式 {code, message, data}
- **错误码**: 0成功, 1xxx通用错误, 2xxx认证错误, 3xxx业务错误
- **鉴权方式**: Bearer Token

## Technical Decisions

| Decision | Rationale |
|----------|-----------|
| 微信原生小程序 | 客户需求，更好的微信生态集成 |
| Node.js后端 | 文档指定，适合高并发场景 |
| MySQL主数据库 | 关系型数据，事务支持 |
| Redis缓存 | 查看机会计数、会话缓存、热点数据 |
| 双用户体系完全隔离 | 防止商业模式漏洞，企业和临工权限完全不同 |
| 所有配置后台可配 | 价格、比例、次数等不能写死在代码中 |
| 信用分自动计算 | 无需人工评价，基于行为自动触发 |
| 临时管理员角色 | 解决现场监管问题，从临工中指定 |
| 查看机会FIFO消耗 | 优先消耗即将过期的机会 |
| 微信服务通知需授权 | 遵循微信规范，用户主动授权 |

## 关键技术难点

### 1. 双用户体系权限隔离
- 企业和临工看到的页面完全不同
- 需要在路由层面做权限控制
- 防止商业模式漏洞

### 2. 临工用工状态机
- 状态流转: 报名→分配→确认→进行中→结算→完成
- 涉及多方确认(工厂、临工、管理员、平台)
- 需要严格的状态机设计

### 3. 微信支付集成
- 企业付款到零钱(工资发放)
- 用户提现
- 订单支付
- 需要处理支付回调和异常情况

### 4. 信用分自动计算
- 基于行为自动计算
- 需要设计触发机制
- 影响用户权限

### 5. 查看机会消耗机制
- 防止重复查看
- 会员免费查看
- 需要Redis缓存优化
- FIFO过期策略

### 6. 实时通知推送
- 站内消息 + 微信服务通知双渠道
- 订阅授权管理
- 定时推送任务

### 7. 地理位置计算
- 用工列表按距离排序
- 需要经纬度计算
- 地图定位功能

### 表结构 (30+张表)
- **用户相关**: users, enterprise_cert, worker_cert
- **供需信息**: info_posts, contact_views
- **用工相关**: job_posts, job_applications, job_checkins, job_photos, job_issues, job_settlements, job_settlement_details
- **财务相关**: view_chances, wallet, wallet_transactions, withdrawals, orders, salary_payments
- **广告相关**: advertisements
- **消息相关**: messages, wechat_subscriptions
- **举报曝光**: reports, blacklist
- **配置相关**: system_config, job_types, open_cities, keywords_blacklist, categories
- **管理员**: admin_users

### 索引策略
- 主键使用自增BIGINT
- 外键字段建立索引
- 常用查询条件字段建立索引
- 状态字段建立索引
- 时间字段建立索引

### Redis缓存
- 会话缓存: session:{token}
- 查看机会计数: view_chances:{userId}
- 未读消息数: unread_msg:{userId}
- 热点配置: config:{configKey}, job_types, open_cities
