# E2E 测试文档

## 概述
本文档描述小灵通项目的端到端（E2E）测试实现方案。

## 测试框架
- **框架**: Playwright
- **语言**: TypeScript
- **测试运行器**: Playwright Test

## 测试覆盖范围

### 1. 认证模块 (4 个测试)
- 用户注册
- 用户登录
- 登录失败处理
- Token 刷新

### 2. 发布模块 (5 个测试)
- 创建发布
- 搜索发布
- 按分类筛选
- 获取发布详情
- 更新和删除发布

### 3. 支付模块 (4 个测试)
- 创建支付订单
- 获取钱包余额
- 查看交易记录
- 使用豆子解锁发布

### 4. 用户交互 (7 个测试)
- 添加收藏
- 获取收藏列表
- 发送消息
- 获取消息列表
- 用户评分
- 获取用户资料
- 更新用户资料

## 运行测试

### 运行所有 E2E 测试
```bash
npm run test:e2e
```

### 运行特定测试文件
```bash
npx playwright test test/e2e/auth.e2e-spec.ts
```

### 以 UI 模式运行
```bash
npx playwright test --ui
```

### 生成 HTML 报告
```bash
npx playwright show-report
```

## 测试数据准备

在运行 E2E 测试前，需要准备测试数据库和初始数据。

## 最佳实践

1. 使用 Fixtures 管理测试数据和依赖
2. 每个测试应该是独立的
3. 使用有意义的测试名称
4. 避免测试之间的依赖关系
5. 定期检查和更新测试用例
