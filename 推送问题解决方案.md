# 🚨 推送问题诊断与解决方案

## 问题诊断

### 错误信息
```
fatal: unable to access 'https://github.com/SheltonMark/xiaolingtong.git/':
schannel: failed to receive handshake, SSL/TLS connection failed
```

### 根本原因
- Windows schannel（SSL后端）与Clash Verge代理的SSL证书冲突
- 命令行git无法通过代理完成SSL握手
- 这是Windows + Clash Verge + Git的已知兼容性问题

### 已尝试的方法（均失败）
- ❌ 配置http.proxy
- ❌ 禁用SSL验证
- ❌ 更换代理端口
- ❌ 配置credential helper

### 方案4: 使用SSH方式（需要配置SSH密钥）⭐⭐
**成功率**: 80%
**耗时**: 15分钟（首次配置）
**难度**: ⭐⭐⭐

#### 前提条件
需要先在GitHub配置SSH密钥

#### 操作步骤
1. **生成SSH密钥**（如果没有）
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```

2. **添加到GitHub**
   - 复制公钥: `cat ~/.ssh/id_ed25519.pub`
   - 访问: https://github.com/settings/keys
   - 点击 "New SSH key"
   - 粘贴公钥

3. **修改远程URL**
   ```bash
   git remote set-url origin git@github.com:SheltonMark/xiaolingtong.git
   ```

4. **推送**
   ```bash
   git push -u origin main
   ```
