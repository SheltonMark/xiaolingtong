# 测试模式使用说明

## 功能说明
测试模式允许你使用单个微信号模拟不同用户，用于测试解锁联系方式、在线聊天等功能。

## 使用步骤

### 1. 启用测试模式
编辑 `app.js` 文件，修改以下配置：

```javascript
globalData: {
  // ...其他配置...

  // ===== 测试模式开关 =====
  TEST_MODE: true,           // 改为 true 启用测试模式
  TEST_USER_ID: 999          // 设置要模拟的用户ID
  // ========================
}
```

### 2. 获取用户ID
你需要知道要模拟的用户ID。可以通过以下方式获取：
- 查看数据库 `users` 表
- 在"我的"页面查看用户信息（需要后端返回）
- 查看发布信息的 userId 字段

### 3. 测试场景

#### 场景A：测试查看别人发布的信息
1. 用你的真实账号发布一条信息（记住信息ID）
2. 启用测试模式，设置 `TEST_USER_ID` 为另一个用户ID（如999）
3. 刷新小程序，此时你的 userId 会被替换为 999
4. 查看刚才发布的信息，测试解锁功能

#### 场景B：测试在线聊天
1. 用真实账号发布信息
2. 启用测试模式，模拟另一个用户
3. 点击"在线聊"按钮，测试是否提示需要解锁
4. 解锁后再次点击，测试是否能正常进入聊天

#### 场景C：测试自己不能和自己对话
1. 用真实账号发布信息
2. 不启用测试模式（或设置 TEST_USER_ID 为你的真实ID）
3. 点击"在线聊"，应该提示"不能和自己对话"

### 4. 关闭测试模式
**测试完成后务必关闭测试模式！**

```javascript
globalData: {
  TEST_MODE: false,          // 改回 false
  TEST_USER_ID: null         // 改回 null
}
```

## 注意事项

⚠️ **重要提醒**：
1. 测试模式只影响前端的 userId 判断，不影响后端API调用
2. 后端API仍然使用真实的token和userId
3. 测试完成后**必须关闭测试模式**，否则会导致逻辑混乱
4. 不要在生产环境启用测试模式
5. 测试模式启用时，控制台会显示警告信息

## 常见问题

**Q: 启用测试模式后，为什么还是显示我的信息？**
A: 检查 TEST_USER_ID 是否设置正确，刷新小程序后查看控制台是否有警告信息。

**Q: 测试模式下能正常解锁吗？**
A: 可以，但解锁记录会关联到你的真实userId，不是模拟的userId。

**Q: 如何恢复正常？**
A: 将 TEST_MODE 改为 false，重新编译小程序即可。
