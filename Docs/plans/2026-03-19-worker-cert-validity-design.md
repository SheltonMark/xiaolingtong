# 临工认证身份证有效期展示设计

## 目标

在临工认证页增加一条“身份证有效期”展示与可编辑输入项。

## 范围

- 仅修改小程序前端 `pages/cert-worker/*`
- 复用现有身份证反面 OCR 返回的 `validDate`
- 不新增后端字段，不修改数据库

## 方案

1. 在临工认证页表单中新增 `form.idValidity`
2. 身份证反面上传识别后，将 OCR 返回的 `validDate` 回填到 `form.idValidity`
3. 页面增加一条“身份证有效期”输入框，允许用户手动修正
4. 识别到“长期”时按原样展示，不拆分成单独日期组件

## 取舍

- 推荐保留为前端展示字段，先满足审核与录入体验
- 不做日期组件和长期勾选，避免引入额外交互复杂度
- 暂不落库存储，后续如后台审核确实需要再补实体和接口
