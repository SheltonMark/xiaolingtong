# 风险板块定制与内容安全整改设计

## 背景

当前小程序的“曝光”能力包含公开列表、公开详情、发布、评论与分享。甲方仍需保留该业务能力，但前台展示需要弱化为更中性的“风险”板块，同时补齐微信审核关注的内容安全校验闭环。

## 已确认范围

1. 保留现有 `Exposure` 数据模型与三类分类编码：`false_info`、`fraud`、`wage_theft`。
2. 分类展示名称改为后台可配置，前台列表 tab、发布页可选分类、卡片头像首字统一读取该配置。
3. 底部 tab 不再显示“曝光”，改成更中性的“风险”。
4. 内容安全补齐到真实可发布场景：
   - 昵称更新
   - 默认联系资料更新
   - 聊天文本 / 图片发送
   - 头像上传失败时不再本地兜底显示未过检图片

## 设计决策

### 1. 后台配置方案

复用现有 `sys_configs`，新增 3 个配置项：

- `exposure_category_false_info_label`
- `exposure_category_fraud_label`
- `exposure_category_wage_theft_label`

默认值分别为“虚假信息”“欺诈行为”“欠款欠薪”。后台系统配置页天然可编辑，无需新增后台页面。

### 2. 前台读取方案

在 `exposure` 模块新增公开接口 `GET /exposures/settings`，返回前台需要的分类配置：

- 分类 key
- 分类 label
- 分类头像首字

前台页面只依赖该接口，不直接访问后台配置接口。

### 3. 前台展示调整

- 底部 tab 文案改为“风险”
- 风险列表页标题与说明文案改为中性描述
- 发布页增加“风险类型”选择
- 风险卡片头像字改为配置名称的第一个字
- 收藏/我的页面里 `targetType=exposure` 的展示文案改为“风险”

### 4. 内容安全补齐

后端统一继续复用 `WechatSecurityService.assertSafeSubmission`：

- `UserService.updateProfile` 校验昵称/手机号文本
- `UserService.updateDefaultContactProfile` 校验联系人、手机号、微信号、微信二维码
- `ChatService.sendMessage` 校验聊天文本与图片 URL

前端头像上传若失败，只提示失败，不再把本地临时图写入缓存。

## 风险与边界

1. 本次不改数据库分类编码，避免历史数据迁移。
2. 聊天语音仍沿用现有消息结构，本次只补发送入口的文本/图片内容安全调用，不重构音频审核链路。
3. 后台“曝光管理”命名不变，仅小程序前台做中性化处理。
