# 后台设置主管最小实现计划

更新时间：2026-03-23
适用分支：`zyc_main`

## 目标

- 主管只能由后台指定，不再由企业前台设置。
- 继续复用现有 `job_applications.isSupervisor` 字段，不新增表结构。
- 企业端“查看报名”只展示当前主管结果，不提供设置入口。
- 考勤、签到、结算继续以 `isSupervisor` 作为主管权限来源。

## 现状结论

| 模块 | 当前情况 | 结论 |
| --- | --- | --- |
| 数据结构 | `job_applications.isSupervisor` 已存在 | 可直接复用，无需改表 |
| 考勤权限 | `WorkService.ensureSupervisor` 已依赖 `isSupervisor` | 权限链路已具备 |
| 企业前台 | 原有 `POST /jobs/:id/supervisor` 能设置主管 | 需禁用企业侧入口 |
| 后台管理 | `admin.html` 在“用工订单分配”里可顺带指定主管 | 需补独立“保存主管”能力 |
| 企业展示 | 报名页仅在报名项上显示 `主管` 标签 | 需补“当前主管/待后台指派”汇总展示 |

## 实施范围

| 类别 | 改动 | 说明 |
| --- | --- | --- |
| 文档 | 新增本计划文档 | 固定实现边界与验收项 |
| 后端 | 新增后台设置主管接口 | `POST /admin/job-orders/:id/supervisor` |
| 后端 | 复用统一主管设置逻辑 | 后台初次分配与独立设置走同一套校验 |
| 后端 | 禁用企业端设置主管接口 | 企业端接口改为明确拒绝 |
| 后端 | 统一提示文案 | “临工管理员”收口为“主管/后台设置主管” |
| 后台 | 用工订单详情支持独立保存主管 | 非招工中状态也可调整主管 |
| 前端 | 企业端报名页显示主管结果 | 只读展示“当前主管/待后台指派” |

## 验收标准

- 后台可对指定招工单独立设置主管，不依赖初次分配动作。
- 被设置主管后，原主管标记被清空，新主管标记生效。
- 企业端无法再通过前台接口设置主管。
- 企业端“查看报名”页面可看到“当前主管：xxx”或“待后台指派”。
- 没有主管时，签到/考勤提示为后台未设置主管。
