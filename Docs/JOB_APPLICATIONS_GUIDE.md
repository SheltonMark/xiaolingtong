# 招工详情管理界面 - 使用指南

## 功能概述

企业端可以在招工管理页面查看每个招工的详细信息，包括所有报名者的列表，并可以接受或拒绝报名者的应用。

## 页面结构

### 1. 招工基本信息区
显示招工的核心信息：
- 招工标题
- 招工状态（招工中、已满员、进行中、待结算、已结算、已关闭）
- 工价（元/天或元/件）
- 人数需求（需X人 · 已报名Y人）
- 工作日期

### 2. 报名者管理区
按状态分类显示所有报名者：

#### 待审核 (Pending)
- 显示报名者姓名、信用分、订单数
- 提供"接受"和"拒绝"两个按钮
- 点击按钮后显示确认对话框

#### 已接受 (Accepted)
- 显示报名者信息
- 状态徽章显示"待确认"
- 等待报名者确认出勤

#### 已确认 (Confirmed)
- 显示报名者信息
- 状态徽章显示"已入选"
- 报名者已确认出勤，可以进行后续操作

#### 已拒绝 (Rejected)
- 显示被拒绝的报名者信息
- 状态徽章显示"已拒绝"
- 仅供查看，无法进行操作

## 使用流程

### 1. 进入招工详情
- 在招工管理页面点击某个招工项目
- 系统自动跳转到招工详情管理页面

### 2. 查看报名者
- 页面加载时自动获取该招工的所有报名者
- 报名者按状态分组显示
- 每个报名者显示基本信息（姓名、信用分、订单数）

### 3. 接受报名
- 在"待审核"分组中找到要接受的报名者
- 点击"接受"按钮
- 确认对话框出现，点击"确定"确认
- 报名者状态变为"已接受"，移到"已接受"分组
- 报名者会收到通知

### 4. 拒绝报名
- 在"待审核"分组中找到要拒绝的报名者
- 点击"拒绝"按钮
- 确认对话框出现，点击"确定"确认
- 报名者状态变为"已拒绝"，移到"已拒绝"分组
- 报名者会收到通知

## 状态转换流程

```
报名者报名
    ↓
pending (待审核)
    ↓
├─→ 企业接受 → accepted (已接受) → 报名者确认 → confirmed (已确认)
│
└─→ 企业拒绝 → rejected (已拒绝)
```

## API 端点

### 获取招工的所有报名者
```
GET /jobs/:jobId/applications
```

**响应格式**:
```json
{
  "pending": [
    {
      "id": 1,
      "status": "pending",
      "worker": {
        "id": 1,
        "name": "张三",
        "creditScore": 95,
        "totalOrders": 10
      }
    }
  ],
  "accepted": [...],
  "confirmed": [...],
  "rejected": [...]
}
```

### 接受或拒绝报名
```
POST /jobs/:jobId/applications/:applicationId/accept
```

**请求体**:
```json
{
  "action": "accepted" // 或 "rejected"
}
```

**响应**: 更新后的应用对象

## 权限控制

- 只有招工的发布者（企业）可以接受/拒绝报名
- 只有待审核状态的报名可以被接受或拒绝
- 其他状态的报名无法进行操作

## 常见问题

### Q: 为什么看不到某个报名者？
A: 报名者可能处于其他状态（已接受、已确认、已拒绝）。请查看对应的分组。

### Q: 接受后可以取消吗？
A: 接受后报名者状态变为"已接受"，此时无法直接拒绝。需要等待报名者确认或取消。

### Q: 报名者信息中的信用分和订单数是什么？
A:
- 信用分：报名者的信用评分，范围 0-100
- 订单数：报名者完成的工作订单总数
