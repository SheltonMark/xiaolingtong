# 小灵通页面手工回归清单

更新时间：2026-03-22
适用分支：`fix/issue-batch-20260322`
基线提交：`e96201e`

## 回归目标

本轮重点覆盖以下联动：

1. 招工录用、拒绝后的系统通知。
2. 企业端报名管理状态与实际流程阶段一致性。
3. 评价企业页面的打分与提交流程。
4. 考勤汇总与结算页的秒级时间展示。
5. 聊天页每条消息时间展示。
6. 我的发布按钮去重与主动作收敛。
7. 系统通知点击后的页面跳转。

## 账号准备

- 企业账号 1 个
- 临工账号 1 个
- 至少 1 条可报名招工
- 至少 1 条已录用并可进入考勤的招工
- 至少 1 条已进入结算阶段的招工

## 回归清单

### 1. 录用通知

- 企业端进入“用工流程 > 报名管理”，录用一名临工。
- 临工端进入“消息中心 > 系统通知”。
- 预期：
  - 出现“报名已录用”通知。
  - 点击通知进入 [my-applications](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/my-applications/my-applications.wxml) 对应页面。
  - 该岗位状态显示为“待出勤”。

### 2. 拒绝通知

- 企业端拒绝一名临工报名。
- 临工端查看系统通知。
- 预期：
  - 出现“报名未通过”或“报名已拒绝”类通知。
  - 点击通知进入“我的报名”页面。
  - 岗位状态显示为“未通过”。

### 3. 企业端报名状态

- 在同一招工下准备 `pending / accepted / confirmed / working / done / rejected` 多种报名状态数据。
- 企业端进入 [job-process](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/job-process/job-process.wxml)。
- 预期：
  - 筛选标签分别显示“待审核 / 已录用 / 已确认 / 进行中 / 已完工 / 已拒绝”。
  - 切换筛选后只出现对应状态数据。
  - 概览统计和列表数量一致。

### 4. 评价企业页面

- 临工端从“我的报名”或“工资结算”进入 [rate](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/rate/rate.wxml)。
- 点击不同评分项的 1-5 星。
- 输入评价内容并提交。
- 预期：
  - 点星后对应评分即时变化。
  - 提交按钮有 loading 状态。
  - 成功时出现“评价成功”。
  - 异常时能看到错误提示，不再静默失败。

### 5. 考勤汇总秒级时间

- 主管完成签到和签退，确保签到时间与签退时间都带秒。
- 企业端进入 [job-process](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/job-process/job-process.wxml) 的“考勤汇总”。
- 企业端再进入 [settlement](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/settlement/settlement.wxml) 的“考勤汇总”。
- 预期：
  - 两个页面都显示 `HH:mm:ss`。
  - 同一工人的签到、签退时间在两个页面保持一致。
  - 列对齐稳定，不出现错位。

### 6. 聊天消息时间

- 在聊天会话中准备历史消息、自己发送消息、对方发送消息。
- 打开 [chat](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/chat/chat.wxml)。
- 预期：
  - 每一条消息气泡下方都有时间。
  - 自己的时间右对齐，对方时间左对齐。
  - 实时收到新消息后也显示时间。

### 7. 我的发布按钮

- 企业端进入 [my-posts](C:/Users/15700/Desktop/work/project/小灵通/.worktrees/fix-issue-batch-20260322/pages/my-posts/my-posts.wxml)。
- 检查不同招工状态卡片。
- 预期：
  - 不再出现两个语义重复的“查看”按钮。
  - 招工主流程按钮优先展示“查看报名 / 去考勤 / 去结算 / 管理招工”等动作。
  - 详情按钮对招工显示为“岗位详情”。

### 8. 系统通知跳转

- 依次点开以下通知：
  - 新的临工报名
  - 报名已接受
  - 报名已拒绝
  - 工作即将开始
  - 工作结算完成
  - 企业评价提醒
- 预期：
  - 每条通知都能跳转到真实页面，而不是无效路径。
  - 报名类通知跳到 `job-process` 或 `my-applications`。
  - 结算和评价提醒跳到 `settlement` 或 `job-process` 对应页签。
